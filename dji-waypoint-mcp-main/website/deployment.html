部署指南

详细的安装和配置指南，帮助您快速部署大疆航线规划MCP服务。

环境要求

🐍 Python环境

  • Python 3.8 或更高版本
  • pip 包管理器
  • 虚拟环境 (推荐)

💻 操作系统

  • macOS 10.14+
  • Ubuntu 18.04+
  • Windows 10+
  • CentOS 7+

📦 系统依赖

  • GDAL/OGR (地理数据处理)
  • PROJ (坐标转换)
  • libxml2 (XML处理)
  • Git (版本控制)

💾 硬件要求

  • 内存: 最少 2GB RAM
  • 存储: 最少 1GB 可用空间
  • CPU: 双核处理器
  • 网络: 稳定的网络连接

环境检查

# 检查Python版本
python --version
# 应该显示 Python 3.8.x 或更高

# 检查pip版本
pip --version

# 检查Git
git --version

# 检查系统依赖 (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install -y python3-dev libgdal-dev libproj-dev libxml2-dev

# 检查系统依赖 (macOS)
brew install gdal proj libxml2

# 检查系统依赖 (CentOS/RHEL)
sudo yum install -y python3-devel gdal-devel proj-devel libxml2-devel

安装步骤

1

获取源代码

# 克隆项目仓库
git clone https://github.com/your-org/dji-waypoint-mcp.git
cd dji-waypoint-mcp

# 查看项目结构
ls -la
2

创建虚拟环境

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Linux/macOS:
source venv/bin/activate
# Windows:
venv\Scripts\activate

# 验证虚拟环境
which python
3

安装依赖包

# 升级pip
pip install --upgrade pip

# 安装核心依赖
pip install -r requirements.txt

# 安装开发依赖 (可选)
pip install -r requirements-dev.txt

# 验证安装
pip list | grep -E "(mcp|pydantic|numpy|shapely)"
4

配置环境变量

# 创建环境配置文件
cp .env.example .env

# 编辑配置文件
nano .env

# 设置Python路径
export PYTHONPATH=$PYTHONPATH:$(pwd)/src

# 设置日志级别
export LOG_LEVEL=INFO

# 设置输出目录
export OUTPUT_DIR=./output
5

运行测试

# 运行完整测试套件
python test_comprehensive.py

# 查看测试报告
cat test_report.md

# 运行单元测试
python -m pytest tests/unit/ -v

# 检查代码覆盖率
python -m pytest --cov=src/dji_waypoint_mcp tests/

配置说明

服务器配置

在 src/dji_waypoint_mcp/config.py 中配置服务器参数：

# 服务器基本信息
SERVER_NAME = "dji-waypoint-mcp"
SERVER_VERSION = "1.0.0"

# 文件路径配置
OUTPUT_DIR = Path("./output")
TEMP_DIR = Path("./temp")

# 日志配置
LOG_LEVEL = "INFO"
LOG_FORMAT = "%(asctime)s - %(name)s - %(levelname)s - %(message)s"

# 性能配置
ENABLE_CACHING = True
CACHE_SIZE = 1000
WORKER_THREADS = 4

# 安全配置
MAX_WAYPOINTS_PER_MISSION = 999
MAX_CONCURRENT_MISSIONS = 10

Docker部署

1. 构建Docker镜像

# 创建Dockerfile
cat > Dockerfile << 'EOF'
FROM python:3.9-slim

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    libgdal-dev \
    libproj-dev \
    libxml2-dev \
    git \
    && rm -rf /var/lib/apt/lists/*

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制源代码
COPY src/ ./src/
COPY mcp/ ./mcp/
COPY run_server.py .

# 创建输出目录
RUN mkdir -p output temp logs

# 设置环境变量
ENV PYTHONPATH=/app
ENV LOG_LEVEL=INFO

# 暴露端口 (如果需要)
EXPOSE 8080

# 启动命令
CMD ["python", "run_server.py"]
EOF

# 构建镜像
docker build -t dji-waypoint-mcp:latest .

2. 运行Docker容器

# 运行容器
docker run -d \
  --name dji-waypoint-mcp \
  -v $(pwd)/output:/app/output \
  -v $(pwd)/logs:/app/logs \
  -e LOG_LEVEL=INFO \
  dji-waypoint-mcp:latest

# 查看容器状态
docker ps

# 查看容器日志
docker logs dji-waypoint-mcp

# 进入容器调试
docker exec -it dji-waypoint-mcp bash

3. Docker Compose部署

# 创建docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  dji-waypoint-mcp:
    build: .
    container_name: dji-waypoint-mcp
    environment:
      - LOG_LEVEL=INFO
      - OUTPUT_DIR=/app/output
      - TEMP_DIR=/app/temp
    volumes:
      - ./output:/app/output
      - ./logs:/app/logs
      - ./temp:/app/temp
    restart: unless-stopped
    networks:
      - mcp-network

networks:
  mcp-network:
    driver: bridge
EOF

# 启动服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

生产环境部署

系统服务配置

创建systemd服务文件：

# 创建服务文件
sudo tee /etc/systemd/system/dji-waypoint-mcp.service << 'EOF'
[Unit]
Description=DJI Waypoint MCP Service
After=network.target

[Service]
Type=simple
User=mcp
Group=mcp
WorkingDirectory=/opt/dji-waypoint-mcp
Environment=PYTHONPATH=/opt/dji-waypoint-mcp
Environment=LOG_LEVEL=INFO
ExecStart=/opt/dji-waypoint-mcp/venv/bin/python run_server.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 重新加载systemd
sudo systemctl daemon-reload

# 启用服务
sudo systemctl enable dji-waypoint-mcp

# 启动服务
sudo systemctl start dji-waypoint-mcp

# 查看服务状态
sudo systemctl status dji-waypoint-mcp

日志轮转配置

# 创建logrotate配置
sudo tee /etc/logrotate.d/dji-waypoint-mcp << 'EOF'
/opt/dji-waypoint-mcp/logs/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 644 mcp mcp
    postrotate
        systemctl reload dji-waypoint-mcp
    endscript
}
EOF

监控配置

# 创建健康检查脚本
cat > /opt/dji-waypoint-mcp/health_check.sh << 'EOF'
#!/bin/bash

# 检查服务状态
if ! systemctl is-active --quiet dji-waypoint-mcp; then
    echo "Service is not running"
    exit 1
fi

# 检查进程
if ! pgrep -f "run_server.py" > /dev/null; then
    echo "Process not found"
    exit 1
fi

# 检查日志错误
if tail -n 100 /opt/dji-waypoint-mcp/logs/dji_waypoint_mcp.log | grep -q "ERROR"; then
    echo "Errors found in logs"
    exit 1
fi

echo "Service is healthy"
exit 0
EOF

chmod +x /opt/dji-waypoint-mcp/health_check.sh

# 添加到crontab进行定期检查
echo "*/5 * * * * /opt/dji-waypoint-mcp/health_check.sh" | crontab -

故障排除

🔧 常见问题

问题: 导入模块失败

症状: ModuleNotFoundError: No module named 'src.dji_waypoint_mcp'

解决方案:

# 确保Python路径正确
export PYTHONPATH=$PYTHONPATH:$(pwd)/src

# 或者使用绝对路径
export PYTHONPATH=/path/to/dji-waypoint-mcp/src

# 验证路径设置
echo $PYTHONPATH

问题: 依赖包安装失败

症状: pip install 失败，特别是 GDAL 相关包

解决方案:

# Ubuntu/Debian
sudo apt-get install -y python3-dev libgdal-dev libproj-dev

# macOS
brew install gdal proj

# CentOS/RHEL
sudo yum install -y python3-devel gdal-devel proj-devel

# 然后重新安装Python包
pip install --upgrade pip
pip install -r requirements.txt

问题: MCP客户端连接失败

症状: MCP客户端无法连接到服务

解决方案:

# 检查服务是否正常启动
python run_server.py

# 检查配置文件格式
python -c "import json; json.load(open('.kiro/settings/mcp.json'))"

# 检查权限
chmod +x run_server.py

# 检查环境变量
env | grep PYTHON

问题: 坐标转换不准确

症状: 坐标转换结果偏差较大

解决方案:

# 检查坐标系统设置
# 中国境内使用GCJ02，境外使用WGS84

# 验证坐标范围
# 纬度: -90 到 90
# 经度: -180 到 180

# 检查PROJ数据文件
python -c "import pyproj; print(pyproj.datadir.get_data_dir())"

问题: KMZ文件无法导入DJI Pilot 2

症状: 生成的KMZ文件在DJI Pilot 2中无法识别

解决方案:

# 检查文件格式和WPML标准兼容性
unzip -l output/mission.kmz

# 验证航点数量限制
# 确保航点数量不超过99个

# 检查文件编码
file output/mission.kmz

# 验证XML格式
xmllint --noout waylines.wpml
